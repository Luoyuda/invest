# A 股财报与公告解读

## Goal

针对单个 A 股公告或财报做事实拆解和影响判断，区分短线情绪、中期业绩、长期基本面，不给交易指令。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`

## Step 1: 确认公告对象

必须确认：

- 证券简称、代码、交易所。
- 公告标题。
- 披露日期。
- 披露主体。
- 原始公告链接。

如果用户只贴摘要或传闻，先查原始公告；找不到原始公告时，只能输出“暂未核验”。

## Step 2: 分类公告类型

| 类型 | 重点 |
|---|---|
| 定期报告 | 收入、利润、扣非、现金流、毛利率、费用率、负债、应收、存货 |
| 业绩预告/快报 | 增减幅、原因、是否符合预期、是否一次性因素 |
| 分红/回购 | 现金流支撑、分红率、回购目的、历史执行 |
| 减持/增持 | 主体、规模、区间、原因、占比 |
| 并购/定增 | 标的质量、估值、商誉、摊薄、审批不确定性 |
| 重大合同/订单 | 金额、周期、毛利、确认节奏、客户集中 |
| 问询/处罚/诉讼 | 监管风险、财务真实性、潜在赔付 |
| ST/退市风险 | 触发规则、时间点、可逆条件 |

## Step 3: 事实摘取

至少输出：

- 公告讲了什么。
- 关键数字和口径。
- 与上一期或市场预期的差异。
- 哪些信息公告没有披露。

所有关键数字必须进入证据包。

## Step 4: 影响链路

分别判断：

- 短线：情绪、资金、预期差、是否已被交易。
- 中期：收入、利润、现金流、订单确认、费用率。
- 长期：竞争格局、行业位置、治理、资产质量。

结论使用：偏利好 / 偏利空 / 影响分化 / 中性 / 暂无法判断。

## Step 5: 输出格式

```markdown
## 【股票简称/代码】公告/财报解读

### 结论
...

### 公告事实
1. ...

### 影响拆解
| 维度 | 判断 | 逻辑 |
|---|---|---|
| 短线 | ... | ... |
| 中期 | ... | ... |
| 长期 | ... | ... |

### 关键风险与未披露信息
1. ...

### 后续观察点
1. ...

### 来源链接
1. ...

以上为公开信息整理，不构成投资建议。
```

## Error Handling

| 情况 | 处理 |
|---|---|
| 没有原始公告 | 不做确定性利好/利空判断 |
| 只有业绩增速没有原因 | 标注“原因待确认” |
| 一次性收益影响利润 | 拆分经营性与非经营性 |
| 利好和利空并存 | 输出“影响分化”，不要强行单边 |
| 市场已提前大涨 | 说明可能已部分反映预期 |
