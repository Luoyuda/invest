# A 股数据提供与核验

## Goal

为其他投资分析 skill 提供统一的数据获取、来源优先级、运行时检查和降级策略。这个 skill 不做投资判断，只交付可追溯数据和证据包。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`

## Step 1: 识别数据需求

将请求归类为以下一种或多种：

| 需求 | 输出 |
|---|---|
| 标的确认 | 证券简称、代码、交易所、是否唯一 |
| 行情 | 价格、价格口径、涨跌幅、成交额、市值、估值、数据时间 |
| 公告 | 标题、披露主体、披露日期、原始链接、摘要 |
| 财报 | 报告期、营收、归母净利润、扣非、毛利率/净利率、现金流、负债 |
| 资金 | 北向、主力资金、融资融券、龙虎榜、成交量/换手 |
| 宏观/行业 | 政策来源、发布日期、影响行业、传导链路 |
| 交易日历 | 当前是否交易日、最近交易日、事件日期 |

默认市场为 A 股。若出现同名公司或跨市场代码，先确认唯一标的。

## Step 2: 运行时检查

如果在 CLI 环境中，先检查基础工具：

```bash
command -v curl >/dev/null && echo CURL_OK || echo CURL_MISSING
command -v python3 >/dev/null && echo PYTHON_OK || echo PYTHON_MISSING
command -v jq >/dev/null && echo JQ_OK || echo JQ_MISSING
```

如果存在可用金融数据 MCP、同花顺 iFinD、TradingView、opencli、浏览器或网页搜索能力，优先使用；否则用官方网页和主流财经源人工核验。

不要因为某个工具不可用就停止，除非无法确认标的身份或关键事实。

## Step 3: 按来源优先级获取数据

优先顺序：

1. 官方/原始来源：交易所、巨潮资讯、监管机构、上市公司公告。
2. 稳定结构化数据源：BaoStock、TuShare Pro、AKShare 中可复核的历史 K 线/财务接口、交易所行情下载。
3. 可核验行情和财经平台：东方财富、同花顺、交易所行情、沪深港通、融资融券。
4. 主流财经媒体：财联社、证券时报、上海证券报、中国证券报、第一财经。
5. 网页抓取、非正式接口和其他来源只作辅助，不作为关键结论唯一依据。

稳定性要求：

- 公告、财报、政策优先使用 S1 原始来源。
- 上一交易日收盘价、历史行情、交易日历优先使用 S2 结构化数据源。
- 盘中实时行情可以使用 S3/S4，但必须保留数据时间；关键推荐最好用至少两个来源交叉校验。
- 如果同一字段多个来源冲突，优先采用稳定性更高、时间更近、口径更明确的来源，并在证据包中记录冲突。

## Step 3.5: 交易时段价格口径

定时任务或自动日报必须先判断运行时间：

- 09:00 前后运行：默认使用上一交易日收盘价或最近可核验价，字段名写 `上一交易日收盘价`，不得写 `当前价`。
- 09:15-09:25：可以输出集合竞价参考价，但必须写 `集合竞价参考，非最终成交价`。
- 09:30-11:30、13:00-15:00：可以输出盘中实时价或延迟行情价，必须写数据源和数据时间。
- 15:00 后：使用当日收盘价。
- 非交易日：使用最近交易日收盘价。

如果行情接口返回的 `data_time` 和任务运行时间不匹配，必须以接口时间为准，并将价格标为 `最近可核验价`。

## Step 4: 生成证据包

每条关键数据都必须转换成 `references/evidence-schema.md` 中的 Evidence Item。至少包含：

- 来源名称。
- 稳定性层级。
- 发布日期或数据时间。
- 链接。
- 支撑的信息点。
- 可靠性等级。
- 限制或缺失。

同时每条关键事实必须转换成 Fact Item，至少包含：

- 来源字段名或原文位置。
- 原始值 `raw_value`。
- 整理值 `normalized_value`。
- 单位 `unit`。
- 换算方式 `transform`。
- 数据时间、报告期或公告日期。

如果无法从来源保留原始值，不得把整理值当作确定事实；只能输出“未取得可审计原始值”。

## Step 5: 输出格式

```markdown
## 数据核验结果

### 标的/范围
- ...

### 可用数据
| 数据项 | 值/摘要 | 数据时间 | 来源 | 可靠性 |
|---|---|---|---|---|

### 缺失或冲突
| 数据项 | 问题 | 处理 |
|---|---|---|

### 证据包
1. E1 ...

### 可交给下游分析的事实
1. F1 ...
```

## Error Handling

| 情况 | 处理 |
|---|---|
| 股票代码无法唯一确认 | 停止并请求补充，不进入分析 |
| 只有新闻没有公告 | 降低可信度，不能替代原始公告 |
| 行情源之间轻微差异 | 标注数据源和时间，不强行统一 |
| 09:00 定时任务需要价格 | 使用上一交易日收盘价或最近可核验价，不写当前价 |
| 关键数据缺失 | 明确缺失，不补编 |
| 来源冲突 | 官方/原始来源优先，并说明冲突 |

## Hard Boundaries

- 不输出买入、卖出、仓位、目标价、止损价。
- 不把数据查询结果包装成投资建议。
- 不使用无法打开或无法定位信息的链接支撑关键事实。
