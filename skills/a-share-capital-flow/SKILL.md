# A 股资金流向分析

## Goal

对 A 股个股、板块或市场方向做资金流向分析，输出可核验的资金事实、统计区间、资金结构和解读边界。资金流是辅助证据，不是买卖建议。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`
- `references/capital-flow.md`

优先协同：

- `a-share-data-provider`
- `a-share-evidence-pack`
- `a-share-stock-analysis`
- `a-share-sector-research`

## IM 输出限制

最终回答最多 5 个 Markdown 表格。超过 5 个表格时，保留核心汇总表，其余明细改为编号列表；来源链接必须用编号列表。发送前可运行 `python3 scripts/validate_answer_format.py /path/to/final-answer.md --max-tables 5`。

## Step 1: 识别资金问题

| 用户问题 | 重点 |
|---|---|
| “大盘资金流怎么样” | 同花顺行业资金与概念资金方向，净流入/净流出前列 |
| “资金流怎么样” | 当日、3/5/10/20 日主力净流入趋势 |
| “主力是不是在流入” | 主力净流入、超大单/大单分解、成交额和价格位置 |
| “为什么涨跌” | 资金与价格、成交额、板块催化是否一致 |
| “北向/融资融券/龙虎榜” | 对应官方或主流来源，不能混用口径 |
| “多个股票资金对比” | 同一来源、同一日期、同一单位下比较 |

默认市场为 A 股。个股必须先确认代码和交易所。

## Step 2: 数据获取

CLI 环境优先使用：

```bash
python3 scripts/fetch_capital_flow.py 300308 --days 20 --output runtime/capital-flow.latest.json
```

多股对比：

```bash
python3 scripts/fetch_capital_flow.py 300308 601899 688981 --days 20 --output runtime/capital-flow.latest.json
```

大盘资金：

```bash
python3 scripts/fetch_capital_flow.py --scope market --provider ths --limit 50 --output runtime/capital-flow.latest.json
```

脚本当前默认 `--provider auto`：东方财富提供 3/5/10/20 日净流入趋势，同花顺补充实时总流入/总流出和大/中/小单拆分。同花顺属于 S4 可选增强源，失败时必须降级，不得阻断任务。
大盘资金当前使用同花顺行业资金和概念资金公开页，输出资金偏好的行业/主题方向；不得写成全市场精确总额。

## Step 3: 必须输出的字段

个股资金流至少包含：

- 标的：名称、代码、交易所。
- 数据时间：当日或最近可核验交易日。
- 当日主力净流入：单位亿元。
- 若同花顺实时字段可用，输出总流入、总流出、净额；若不可用，不得反推。
- 超大单/大单/中单/小单净流入分解：单位亿元。
- 3 日、5 日、10 日、20 日主力净流入合计。
- 来源、接口/页面、访问时间、限制。

如果来源没有提供“主力流入”和“主力流出”总额，不得用净流入反推流入/流出总额。

## Step 4: 解读框架

把事实和推断分开：

- 事实：资金净流入/净流出、订单结构、时间窗口、价格涨跌。
- 推断：资金是否支持行情、是否出现分歧、是否需要降低确定性。
- 反向验证：成交额是否放大、板块是否同步、是否有公告/政策催化。
- 风险：资金流口径依赖单一平台、公开接口可能延迟或字段变化。

## Step 5: 输出格式

```markdown
## 【股票简称/代码】资金流向

### 结论
1-3 句话说明资金面偏强/偏弱/分化/未核验，不给买卖指令。

### 资金事实
| 项目 | 数值 | 口径 | 数据时间 | 来源 |
|---|---:|---|---|---|

### 结构拆分
1. 超大单：...
2. 大单：...
3. 中单：...
4. 小单：...

### 解读边界
- ...

### 来源链接
1. 来源名称，数据时间，链接：...；支撑信息：...

以上为公开信息整理，不构成投资建议。
```

## Error Handling

| 情况 | 处理 |
|---|---|
| 资金接口失败 | 明确“资金面未取得可靠结构化数据”，不补编 |
| 同花顺接口失败 | 保留东方财富趋势数据，标注实时流入/流出未取得 |
| 大盘资金没有总额 | 只写行业/概念资金方向，不得冒充全市场总额 |
| 只有净流入，没有流入/流出总额 | 只写净流入，不反推总额 |
| 多来源冲突 | 标注来源差异，优先官方/更稳定/更新来源 |
| 数据日期不是今天 | 写“最近可核验交易日”，不得冒充今日 |
| 用户要求买卖 | 改为资金风险、观察条件和逻辑失效条件 |

## Quality Checklist

- 代码、交易所和数据时间是否明确。
- 是否写明统计窗口和单位。
- 是否区分当日、3/5/10/20 日。
- 是否把资金事实和推断分开。
- 是否避免无来源的“主力流入/流出”总额。
- 是否保留来源链接和限制。
