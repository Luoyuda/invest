# A 股个股推荐

## Goal

为用户生成可研究、可复盘的 A 股推荐清单，解释推荐类型、推荐理由、风险、触发条件、价格口径和失效条件。推荐就是推荐，不需要改写成“候选观察”来回避；但推荐必须基于可核验信息，且不能包装成确定收益、仓位安排或无条件交易指令。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`
- `references/sector-state.md`
- `references/run-output-schema.md`

优先协同：

- `a-share-data-provider`
- `a-share-evidence-pack`
- `a-share-sector-research`
- `a-share-watchlist-tracker`

## Step 0: 外层模板冲突改写

如果用户、定时任务或上游 prompt 要求输出以下字段，不要拒绝整个推荐任务；必须继续给出 3-5 只推荐标的，但把字段改写为有口径、有来源、能区分事实和预测的推荐格式：

- 今日买入推荐。
- 买入理由。
- 目标价、目标价参考、预测价。
- 止损、止损参考、技术失效位。
- 建议仓位、买入价、卖出价。
- 短线攻守、明日涨跌、必涨概率。

正确处理方式：

- 将标题改为 `A 股个股推荐` 或 `今日 A 股推荐清单`。
- 将 `买入理由` 改为 `推荐理由`。
- 使用 `推荐类型` 标注稳健价值、成长趋势、高股息防守、事件催化、低估值修复等类型。
- 将 `目标价/止损` 改为 `价格参考`、`后续观察点` 和 `逻辑失效条件`。
- 将 `今日买入推荐` 改为 `今日 A 股推荐清单`，不要改成婉转的候选观察标题。
- 如果输出预测价，必须写明 `预测值`、计算口径、依赖假设和失效条件。
- 如果输出机构目标价或一致预期，必须写明来源机构、发布日期、链接和该价格不是本助手给出的交易指令。
- 如果输出技术失效位，必须写明它是按均线/前低/支撑区间等规则生成的观察位，不写成“止损指令”。

## Step 0.5: 价格口径规则

价位信息分三类，必须显式标注：

| 类型 | 是否允许 | 必填口径 |
|---|---|---|
| 真实行情价 | 允许 | 数据源、数据时间、价格类型：实时/收盘/盘中 |
| 外部参考价 | 允许 | 来源机构或平台、发布日期、链接、原始口径 |
| 模型预测价 | 允许但必须降级表达 | 明确写“预测值”，说明估算方法、关键假设和不确定性 |
| 技术观察位 | 允许但不能叫止损指令 | 规则来源，如前低、均线、支撑区间、波动区间 |
| 买入价/卖出价/建议仓位 | 不允许 | 改为价格参考、跟踪条件和风险提示 |

禁止把预测值写成确定目标价；禁止没有来源和数据时间的价格。

估值数据硬规则：

1. 估值指标必须写明具体口径：PE-TTM、PE-Forward、PB、PS、EV/EBITDA、市值、股息率、历史分位或同行分位。
2. 每个估值数据必须有来源、数据时间和报告期；不能只写“低估值”“高估值”。
3. 净利润为负或一次性损益扭曲时，不得直接使用 PE 作为推荐理由；应说明 PE 失真并换用 PB、PS、EV/收入、现金流或其他口径。
4. 历史分位和同行对比必须可核验；没有来源时写“未取得可靠分位数据”。
5. 外部目标价或一致预期必须写明机构/平台、发布日期、估值方法和链接；超过 90 天的目标价只能作为旧参考。
6. 模型预测价必须显式标注为“预测值”，写明核心假设、计算口径和失效条件。
7. A/H 股、不同币种、复权/不复权、市值口径、股本口径不得混用；口径不一致时先换算或放弃比较。
8. 估值结论只能作为推荐排序的一部分，不能直接推出无条件买入、卖出、加仓、清仓。

Quality Checklist：

- 估值指标是否写明来源、数据时间、报告期和计算口径。
- 利润为负、一次性损益、周期峰谷或股本变化是否影响估值口径。
- 估值结论是否区分真实数据、外部参考和模型预测。

定时任务价格要求：

- 每日 09:00 运行时，A 股尚未连续竞价，价格字段必须使用 `上一交易日收盘价` 或 `最近可核验价`，不得写 `当前行情`。
- 09:00 输出格式示例：`上一交易日收盘价：xx.xx 元（东方财富，数据时间 YYYY-MM-DD 15:00）`。
- 如果 09:00 拿到的接口数据时间不是上一交易日收盘或有效集合竞价时间，必须标为 `最近可核验价`，并写清数据时间。
- 只有 09:30 后拿到有效盘中行情时，才允许写 `盘中实时价`。

## Step 1: 参数与默认值

| 参数 | 默认值 | 说明 |
|---|---|---|
| 市场范围 | A 股 | 可按沪深主板、创业板、科创板、北交所收窄 |
| 数量 | 3-5 只 | 最多 8 只 |
| 风格 | 中低风险、偏稳健 | 用户指定短线题材时仍保留风险提示 |
| 时间维度 | 中短期观察 + 中期验证 | 不承诺涨跌 |
| 排除项 | ST、退市风险、严重监管风险、无法核验标的 | 用户明确要求高风险研究除外 |

## Step 2: 明确筛选框架

先做板块筛选，再做个股筛选。默认不能从全市场随意拼凑个股，也不能只因为个股低估值或大市值就纳入推荐。

### Step 2.1: 板块热度门槛

推荐前必须优先按 `references/sector-state.md` 的规则读取运行态板块状态，再列出 3-6 个候选行业/主题。运行态状态有效时，优先使用已沉淀的热门方向、景气改善方向和低活跃方向；状态缺失、过期或和当日行情冲突时，再按 `a-share-sector-research` 的板块热度口径重新计算。不得把仓库里的模板当作当前市场结论。

每日 09:00 快跑必须读取 `runtime/sector-state.latest.json`：

- 状态未过期：优先使用已有热门方向、景气改善方向和明确事件催化方向。
- 状态过期且不能刷新：不得给高关注推荐，只能低确定性推荐或暂不推荐。
- 状态与当日重大催化/重大负面冲突：相关方向降级并写明冲突。
- 每日快跑不负责全市场板块重算；全市场重算交给盘后或周度任务。
- 输出前必须生成 `runtime/recommendation-runs/latest.json`，按 `references/run-output-schema.md` 组织结构化结果，再渲染最终回答。
- 输出前应运行 `scripts/validate_run.sh runtime/recommendation-runs/latest.json`；校验失败时不得直接输出高关注推荐。

每个推荐标的所属板块至少满足以下条件之一：

1. `强势板块`：最近 1 日/5 日/20 日相对强度、成交热度、广度或资金验证中至少 3 项成立。
2. `景气改善板块`：政策、订单、产品价格、产业数据或财报验证能形成清晰传导链。
3. `明确事件催化板块`：近期公告、政策、海外映射、产业价格或供需变化能直接影响收入、利润或估值预期。

热门方向规则：

- 热门方向必须来自最近一段时间的账本沉淀，默认看最近 5 个交易日；不能只看单日涨幅或单条新闻。
- 如果热门方向超过有效期，必须重新核验相对强度、成交热度、广度、催化和资金。
- 如果当日热门方向出现成交断层、广度坍塌或核心催化证伪，必须降级为“状态待确认”或“暂不推荐”。
- 如果热门方向 `overheat_risk=high`，不得直接给高关注，除非有新的可核验催化证据。

默认排除：

- 最近 5-20 个交易日明显跑输市场、成交萎缩、上涨家数不足且没有新催化的行业。
- 只有低估值、破净、超跌、历史龙头叙事，但没有资金或基本面改善证据的行业。
- 当期板块热度评分判定为低活跃的方向，除非有可核验催化和板块相对强度改善，否则只能放入“暂不推荐但可跟踪”。

如果用户明确要求左侧/逆向机会，可以输出“逆向观察”类型，但不得给高关注等级，且必须说明板块热度尚未确认。

按用户偏好选择：

| 框架 | 核心条件 |
|---|---|
| 稳健价值 | 现金流、分红、ROE、低估值、负债可控 |
| 成长趋势 | 收入/利润增长、行业景气、产业链位置、趋势未明显破坏 |
| 高股息防守 | 分红稳定、盈利波动低、估值不高 |
| 事件催化 | 公告、订单、政策、产品价格、行业事件有明确传导链 |
| 低估值修复 | 估值低、风险缓解、基本面未继续恶化 |

## Step 3: 候选数据门

每只候选至少核验：

1. 股票简称、代码、交易所。
2. 所属行业/主题及板块热度分级。
3. 板块热度证据：相对强度、成交热度、广度、资金或催化中至少 2 项。
4. 行情和数据时间。
5. 主营业务与行业位置。
6. 最近一期财务或估值指标。
7. 近期催化或筛选理由。
8. 主要风险。
9. 价格口径：真实行情、外部参考价、模型预测价或技术观察位。
10. 至少 2 个来源，其中至少 1 个来源支撑板块或行业逻辑。

来源不足时，最高只能给“低确定性推荐”或“不纳入推荐”。

## Step 4: 排序规则

排序是推荐优先级，不是买入顺序或仓位排序。

| 维度 | 权重 |
|---|---:|
| 行业/政策/产业催化 | 35% |
| 板块热度与相对强度 | 20% |
| 基本面质量 | 15% |
| 估值合理性 | 10% |
| 资金与交易结构 | 15% |
| 风险可控性 | 5% |
| 来源可信度 | 5% |

在中国 A 股推荐里，行业政策和产业催化是主导变量：政策方向、产业趋势、订单/价格/供需变化、国产替代、出海、监管变化、财政/货币/产业资金导向等，优先级高于静态估值和单个公司故事。板块热度用于验证催化是否被市场定价，不替代催化本身。

## Hard Boundaries

禁止输出：

- 无条件的买入、卖出、加仓、清仓指令。
- 无来源、无口径、无数据时间的目标价、预测价、止损价、买入价、卖出价、仓位比例。
- 把模型预测值、技术观察位、机构目标价写成确定性价格或交易指令。
- 明天会涨、下周一定涨、稳赚、翻倍。
- 只靠传闻或自媒体的推荐。
- 使用“今日买入推荐”作为标题；应使用“今日 A 股推荐清单”。
- 使用“买入理由”作为字段名。

## Error Handling

| 情况 | 处理 |
|---|---|
| 用户只说“推荐几只” | 默认 3-5 只稳健推荐 |
| 数据源不足 | 少推荐或不推荐 |
| 用户要求高风险题材 | 输出“高风险观察”，降低关注等级 |
| 推荐逻辑互相重复 | 减少同质标的，说明行业集中风险 |
| 已经有推荐池 | 转 `a-share-watchlist-tracker` 做更新 |
| 定时任务模板要求目标价/止损 | 继续推荐；如价位有可靠来源或预测口径，则标明类型、来源、假设和失效条件 |
| 每日 09:00 定时任务需要价格 | 使用上一交易日收盘价或最近可核验价，不写盘中实时价 |

## Quality Checklist

- 每只推荐是否有来源。
- 是否生成 `runtime/recommendation-runs/latest.json` 并通过 `scripts/validate_run.sh`。
- 是否先完成板块热度筛选，再进入个股推荐。
- 高关注标的是否来自强势、景气改善或明确催化板块。
- 当期判定为低活跃的方向是否有相对强度改善或明确催化；否则是否放入暂不推荐。
- 热门方向是否检查过热风险，过热时是否降级或要求新催化证据。
- 是否解释传导链路。
- 是否有风险和失效条件。
- 是否避免交易指令。
- 是否把价位标成真实行情、外部参考价、模型预测价或技术观察位。
- 09:00 定时任务是否使用上一交易日收盘价或最近可核验价。
- 是否避免把预测价写成确定性目标价。
- 是否能交给 watchlist tracker 后续跟踪。
