# A股个股候选推荐

## Goal

为用户筛选 A 股候选观察标的，解释为什么这些股票值得关注，以及哪些条件会使推荐逻辑失效。

本 Skill 的“推荐”含义是：生成研究候选池 / 观察清单 / 关注理由，不是买入指令、卖出指令或收益承诺。

## Hard Rules

股票推荐属于高风险金融内容，必须遵守：

- 必须联网核验，不得只凭模型记忆推荐股票。
- 必须给出来源链接，所有关键事实可追溯。
- 不得使用“必涨、稳赚、翻倍、梭哈、抄底、无风险”等表达。
- 不得给明确买入价、卖出价、目标价、止损价、仓位比例。
- 不得替用户做最终交易决定。
- 不得把传闻、群聊、自媒体猜测当成推荐依据。
- 如果来源不足，宁可少推荐或不推荐。
- 输出必须包含风险和推荐逻辑失效条件。

允许输出：

- 候选观察标的。
- 推荐等级：高关注 / 中关注 / 仅观察 / 暂不纳入。
- 关注理由。
- 触发条件和观察点。
- 风险和失效条件。

禁止输出：

- “建议买入 / 建议加仓 / 建议清仓”。
- “目标价 / 止损价 / 建议仓位”。
- “明天会涨 / 下周一定涨”。

## Lessons From Similar Skills

参考同类 Skill 的可取做法，但必须采用更保守的龙虾版本：

- 学习 `A Stock Analyzer`：使用硬筛条件，结合基本面、技术面和量能，不只靠热点。
- 学习 `portfolio management`：按技术面、资金面、基本面、宏观环境分层搜集信息，并强制标注数据时间。
- 学习 `Stock Select`：支持用户自然语言条件选股，例如“主力净流入”“涨停”“低估值高股息”，但不接交易下单能力。

需要规避的做法：

- 不输出机械化目标涨幅、止损价、仓位比例。
- 不把“买卖建议”作为核心交付。
- 不使用需要交易账户、券商账户或个人认证的能力。
- 不依赖单一第三方 API 的黑盒结果，必须解释筛选逻辑。

## Inputs

先识别用户需求类型：

- 市场范围：默认 A 股；如用户指定沪深主板、创业板、科创板、北交所，按用户范围筛。
- 风格偏好：稳健价值、成长趋势、高股息、事件催化、短线题材、低估值修复、防守避险。
- 行业范围：如半导体、机器人、医药、消费、煤炭、银行等。
- 时间维度：短线观察、中线跟踪、长期配置研究。
- 风险偏好：低 / 中 / 高；未知时默认中低风险。
- 数量：默认 3-5 只；最多不超过 8 只。

如果用户只说“推荐几只股票”，默认：

- 输出 3-5 只候选观察标的。
- 偏稳健，不追高位纯题材。
- 不给买卖价位和仓位。

## Research Requirements

每只候选标的至少核验以下信息：

1. 股票简称、代码、交易所。
2. 最近交易日行情：收盘价/涨跌幅/成交额/换手率或量能变化，标注数据时间。
3. 主营业务和行业位置。
4. 最近一期财务核心指标：营收、归母净利润、扣非净利润、ROE、毛利率/净利率、现金流，至少覆盖其中 3 项。
5. 估值或市值维度：PE/PB/PS、市值、股息率、历史分位或同行对比，至少覆盖 1 项。
6. 近期催化：公告、政策、行业景气、产品价格、订单、海外映射、资金流等。
7. 风险：估值、业绩、解禁减持、监管、财务质量、行业价格、竞争格局、流动性。

优先来源：

- 交易所、巨潮资讯、上市公司公告、公司官网投资者关系。
- 证监会、交易所问询函、监管处罚、国家部委/统计机构。
- 东方财富、同花顺、证券时报、上海证券报、中国证券报、财联社、第一财经等可核验财经源。
- 行情和资金数据源必须标注数据时间。

## Screening Frameworks

根据用户风格选择一个或多个框架。

### 1. 稳健价值

适合：低风险、长期跟踪、偏防守用户。

优先筛：

- 主营稳定，现金流较好。
- ROE 稳定，负债可控。
- 分红稳定或股息率有吸引力。
- 估值低于自身历史或行业可比。
- 没有重大监管/财务风险。

排除：

- 业绩大幅下滑但只靠题材上涨。
- 高商誉、高负债、现金流持续恶化。
- 近期大额减持或监管调查。

### 2. 成长趋势

适合：中高风险、关注产业趋势用户。

优先筛：

- 营收或利润保持较高增长。
- ROE、毛利率或现金流质量较好。
- 所在行业有政策或产业景气催化。
- 股价趋势不明显破位，成交量有温和放大。
- 公司在产业链中具备核心环节或客户优势。

参考硬条件，可按市场环境灵活使用：

- 最近一期营收同比为正，优先 > 20%。
- 最近一期归母净利润同比为正，优先 > 20%。
- ROE 优先 > 10%，高质量成长优先 > 15%。
- 股价不明显低于中期均线，或回撤后重新放量修复。

### 3. 高股息/防守

适合：震荡市、防守、现金流偏好用户。

优先筛：

- 股息率较高且分红稳定。
- 盈利波动小，现金流稳定。
- 估值不高，行业需求稳定。
- 政策和信用风险较低。

### 4. 事件催化

适合：短中期主题跟踪，不适合低风险用户。

优先筛：

- 明确公告、订单、政策、产品价格、海外映射。
- 板块有资金关注，但未出现极端拥挤。
- 公司与催化事件有清晰业务关联。

必须说明：

- 催化是否已被市场交易。
- 是短线情绪还是中期业绩逻辑。
- 若催化落空，推荐逻辑如何失效。

## Exclusion Rules

以下标的默认不纳入候选，除非用户明确要求研究高风险标的：

- ST、*ST、退市风险警示。
- 上市未满 1 年且无稳定公开财务记录。
- 最近被立案调查、重大监管处罚且风险未消除。
- 业绩连续大幅亏损且无明确改善证据。
- 高位连续暴涨、严重缩量、明显游资炒作但缺少基本面支撑。
- 关键事实无法核验或来源冲突。

## Ranking Method

候选排序采用“关注优先级”，不是买入排名。

评分维度：

- 基本面质量：25%
- 估值合理性：15%
- 行业/政策/产业催化：20%
- 资金与交易结构：15%
- 风险可控性：15%
- 来源可信度与信息完整性：10%

如果某只股票来源不足，最高只能给“仅观察”，不能给“高关注”。

## Quality Checklist

输出前检查：

- 是否明确这是候选观察清单，不是买入指令。
- 是否每只股票都有来源链接支撑关键事实。
- 是否标注数据时间。
- 是否解释推荐逻辑如何传导到收入、利润、估值或风险偏好。
- 是否说明主要风险和逻辑失效条件。
- 是否避免目标价、止损价、仓位比例和交易指令。
- 是否剔除无法核验、传闻驱动、风险警示和纯炒作标的。
