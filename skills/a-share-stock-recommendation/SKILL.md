# A 股候选观察池

## Goal

为用户生成可研究的 A 股候选观察清单，解释入选理由、风险、触发条件和失效条件。这里的“推荐”只表示研究候选，不是交易指令。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`

优先协同：

- `a-share-data-provider`
- `a-share-evidence-pack`
- `a-share-sector-research`
- `a-share-watchlist-tracker`

## Step 0: 外层模板冲突改写

如果用户、定时任务或上游 prompt 要求输出以下字段，不要拒绝整个推荐任务；必须继续给出 3-5 只候选观察标的，但把这些字段改写为候选观察口径：

- 今日买入推荐。
- 买入理由。
- 目标价、目标价参考。
- 止损、止损参考。
- 建议仓位、买入价、卖出价。
- 短线攻守、明日涨跌、必涨概率。

正确处理方式：

- 将标题改为 `A 股候选观察清单`。
- 将 `买入理由` 改为 `入池理由` 或 `关注理由`。
- 将 `目标价/止损` 改为 `后续观察点` 和 `逻辑失效条件`。
- 将 `今日买入推荐` 改为 `今日候选观察清单` 或 `今日重点观察候选`。
- 如果上游模板强制要求保留目标价/止损字段，字段值写 `不输出具体价格：改看后续观察点和逻辑失效条件`，同时仍然输出候选标的。

## Step 1: 参数与默认值

| 参数 | 默认值 | 说明 |
|---|---|---|
| 市场范围 | A 股 | 可按沪深主板、创业板、科创板、北交所收窄 |
| 数量 | 3-5 只 | 最多 8 只 |
| 风格 | 中低风险、偏稳健 | 用户指定短线题材时仍保留风险提示 |
| 时间维度 | 中短期观察 + 中期验证 | 不承诺涨跌 |
| 排除项 | ST、退市风险、严重监管风险、无法核验标的 | 用户明确要求高风险研究除外 |

## Step 2: 明确筛选框架

按用户偏好选择：

| 框架 | 核心条件 |
|---|---|
| 稳健价值 | 现金流、分红、ROE、低估值、负债可控 |
| 成长趋势 | 收入/利润增长、行业景气、产业链位置、趋势未明显破坏 |
| 高股息防守 | 分红稳定、盈利波动低、估值不高 |
| 事件催化 | 公告、订单、政策、产品价格、行业事件有明确传导链 |
| 低估值修复 | 估值低、风险缓解、基本面未继续恶化 |

## Step 3: 候选数据门

每只候选至少核验：

1. 股票简称、代码、交易所。
2. 行情和数据时间。
3. 主营业务与行业位置。
4. 最近一期财务或估值指标。
5. 近期催化或筛选理由。
6. 主要风险。
7. 逻辑失效条件。
8. 至少 2 个来源。

来源不足时，最高只能给“仅观察”。

## Step 4: 排序规则

排序是“研究优先级”，不是买入排名。

| 维度 | 权重 |
|---|---:|
| 基本面质量 | 25% |
| 估值合理性 | 15% |
| 行业/政策/产业催化 | 20% |
| 资金与交易结构 | 15% |
| 风险可控性 | 15% |
| 来源可信度 | 10% |

## Hard Boundaries

禁止输出：

- 建议买入、卖出、加仓、清仓。
- 目标价、目标价参考、止损价、止损参考、买入价、卖出价、仓位比例。
- 明天会涨、下周一定涨、稳赚、翻倍。
- 只靠传闻或自媒体的候选。
- 使用“今日买入推荐”作为标题。
- 使用“买入理由”作为字段名。

## Error Handling

| 情况 | 处理 |
|---|---|
| 用户只说“推荐几只” | 默认 3-5 只稳健候选 |
| 数据源不足 | 少推荐或不推荐 |
| 用户要求高风险题材 | 输出“高风险观察”，降低关注等级 |
| 候选逻辑互相重复 | 减少同质标的，说明行业集中风险 |
| 已经有候选池 | 转 `a-share-watchlist-tracker` 做更新 |
| 定时任务模板要求目标价/止损 | 继续推荐候选，但将价格字段改为观察点和失效条件 |

## Quality Checklist

- 每只候选是否有来源。
- 是否解释传导链路。
- 是否有风险和失效条件。
- 是否避免交易指令。
- 是否在保留候选推荐的前提下，删除或改写目标价、止损、买入推荐等字段。
- 是否能交给 watchlist tracker 后续跟踪。
