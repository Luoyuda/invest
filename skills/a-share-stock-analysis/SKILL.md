# A 股个股分析

## Goal

对单只或多只 A 股做结构化研究，解释行情、基本面、财务、估值、资金、催化、风险和逻辑失效条件。输出是公开信息分析，不是买卖建议。

必须读取：

- `references/a-share-data-sources.md`
- `references/evidence-schema.md`

优先协同：

- `a-share-data-provider`
- `a-share-evidence-pack`
- `a-share-earnings-announcement-review`（公告/财报专项）
- `a-share-sector-research`（行业专项）

## Step 1: 任务分流

| 用户问题 | 处理方式 |
|---|---|
| “这家公司怎么看” | 使用本 skill |
| “为什么涨/跌” | 本 skill，重点查近 1-5 个交易日事件和资金 |
| “这个公告/财报怎么看” | 转 `a-share-earnings-announcement-review` |
| “这个行业/赛道怎么看” | 转 `a-share-sector-research` |
| 多只股票比较 | 本 skill 的 multi-stock 模板 |

## Step 2: 默认参数

| 参数 | 默认值 |
|---|---|
| 市场 | A 股 |
| 行情时间 | 最近交易日或实时数据，必须标注 |
| 事件窗口 | 近 1-3 个月；涨跌原因看近 1-5 个交易日 |
| 财务 | 最近一期定期报告 |
| 估值 | PE/PB/PS/市值/历史分位或同行对比至少 1 项 |
| 结论 | 偏利好 / 偏利空 / 中性 / 影响分化 / 需要观察 |

## Step 3: 数据门

进入分析前必须确认：

- 证券简称、代码、交易所。
- 最新行情或最近交易日行情及数据时间。
- 主营业务或收入结构。
- 最近一期财务核心指标。
- 估值或市值维度。
- 近期公告/新闻/行业催化。
- 主要风险。

缺少关键项时，结论降级，并说明缺口。

## Step 4: 分析框架

将事实和推断分开：

- 事实：行情、财务、公告、行业、资金。
- 推断：这些事实如何影响收入、利润、估值、风险偏好或资金拥挤度。
- 时间维度：短线情绪、中期业绩、长期产业逻辑。
- 反向假设：什么情况会使判断失效。

## Step 4.5: 估值数据硬规则

涉及估值判断时，必须遵守：

1. 估值指标必须写明具体口径：PE-TTM、PE-Forward、PB、PS、EV/EBITDA、市值、股息率、历史分位或同行分位。
2. 每个估值数据必须有来源、数据时间和报告期；不能只写“估值贵/便宜”。
3. 净利润为负或一次性损益扭曲时，不得直接使用 PE 判断贵便宜；应说明 PE 失真并换用其他口径。
4. 历史分位和同行对比必须可核验；没有来源时写“未取得可靠分位数据”。
5. 外部目标价或一致预期必须写明机构/平台、发布日期、估值方法和链接；超过 90 天的目标价只能作为旧参考。
6. 模型预测价必须显式标注为“预测值”，写明核心假设、计算口径和失效条件。
7. A/H 股、不同币种、复权/不复权、市值口径、股本口径不得混用；口径不一致时先换算或放弃比较。
8. 估值结论只能说明相对历史、同行或盈利预期的高低，不能直接推出买入、卖出、加仓、清仓。

Quality Checklist：

- 估值指标是否写明来源、数据时间、报告期和计算口径。
- 利润为负、一次性损益、周期峰谷或股本变化是否影响估值口径。
- 估值结论是否区分真实数据、外部参考和模型预测。

## Step 5: 输出格式

```markdown
## 【股票简称/代码】个股分析

### 结论
偏利好 / 偏利空 / 中性 / 影响分化 / 需要继续观察。
2-4 句话说明核心判断，不给买卖指令。

### 核心事实
1. 【标的】...
2. 【行情】...
3. 【基本面/财务】...
4. 【估值/资金】...
5. 【公告/行业事件】...

### 涨跌影响逻辑
- 短线：...
- 中期：...
- 长期：...

### 关键观察点
1. ...

### 主要风险
1. ...

### 逻辑失效条件
- ...

### 来源链接
1. 来源名称，发布日期/数据时间，链接：...；支撑信息：...

以上为公开信息整理，不构成投资建议。
```

## Multi-stock Comparison

多股比较时使用：

```markdown
## 多股对比分析

| 股票 | 业务位置 | 近期催化 | 财务质量 | 估值/市值 | 资金关注 | 主要风险 | 结论边界 |
|---|---|---|---|---|---|---|---|

### 共同风险
...

### 来源链接
...
```

不要强行排名；若用户要求排序，必须说明排序依据。

## Error Handling

| 情况 | 处理 |
|---|---|
| 简称重名 | 先确认代码 |
| 关键来源不足 | 不输出高置信结论 |
| 传闻驱动 | 明确未核验，不作为判断依据 |
| 财务/估值缺失 | 标注缺口，降低结论确定性 |
| 用户要求买卖 | 改为风险和观察条件 |

## Quality Checklist

- 代码、主体、交易所是否确认。
- 数据时间是否清楚。
- 来源是否支撑正文事实。
- 是否有风险和逻辑失效条件。
- 末尾是否有来源链接。
