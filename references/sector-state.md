# A 股板块状态账本

本文件定义板块热度和低活跃方向的短周期沉淀规则。它不是永久行业黑名单或永久热门名单，也不应该在仓库里提交“当前市场热门/低活跃板块”的静态结论。真实板块状态应由最近一段时间的定时任务、行情数据和行业研究在运行时滚动生成。

## 1. 使用目标

个股推荐时应优先读取本账本，避免每次从零计算板块强弱，也避免凭印象选择“热门”或“低活跃”方向。

仓库只保存规则、字段和模板；具体的当期板块状态应保存到运行环境的状态文件、数据库、日志或定时任务产物中，并带 `as_of`、`valid_until` 和来源证据。没有有效运行态状态时，必须重新计算，不得把本文件当作当前市场结论。

账本用于沉淀：

- `热门方向`：近期被市场持续定价、成交活跃、有政策/产业/事件催化的方向。
- `短线主线`：热门方向或景气改善方向中，涨幅/相对强度靠前、资金流入或成交热度高，且有产业/政策/事件催化支撑的 2-3 条优先方向。
- `景气改善方向`：基本面或政策预期改善，但市场热度尚未完全扩散的方向。
- `低活跃方向`：一段时间内跑输市场、成交萎缩、广度不足或缺少催化的方向。
- `逆向观察方向`：预期低位但尚未获得资金和热度确认的方向。

## 2. 生命周期

默认有效期为 5 个交易日。每日 09:00 定时任务、盘后复盘或周度自检可以更新账本。

更新规则：

1. 每日任务只做增量更新：保留最近 5 个交易日的证据，不因单日波动直接改写状态。
2. 周度任务做完整刷新：重新计算 1 日、5 日、20 日相对强度、成交热度、广度、催化和资金。
3. 出现重大政策、产业价格、订单、监管或海外映射事件时，可以提前刷新相关方向。
4. 超过 5 个交易日未更新的状态必须标记为 `stale`，不能作为高关注推荐依据。
5. 状态变化必须保留原因：从热门降级、从低活跃上调、从景气改善转强势都要写明触发证据。

状态过期后的硬处理：

- `valid_until` 已过期且无法刷新时，不得给高关注推荐。
- 当日证据与账本状态冲突时，相关方向降级为 `unknown` 或低确定性推荐。
- 出现重大利空、催化证伪、成交断层、广度坍塌时，热门方向必须降级。
- `overheat_risk=high` 时，不得直接给高关注，除非有新的可核验催化证据。

## 3. 状态判定

| 状态 | 判定条件 | 推荐使用方式 |
|---|---|---|
| 热门方向 | 最近 5 个交易日相对强度、成交热度、广度、催化、资金中至少 3 项持续成立 | 可作为优先推荐池 |
| 景气改善方向 | 政策、供需、价格、订单、财报或产业数据改善，市场热度尚未完全确认 | 可推荐，但需要说明验证指标 |
| 低活跃方向 | 最近 5-20 个交易日跑输市场、成交萎缩、上涨家数不足，且缺少新催化 | 默认不作为推荐池 |
| 逆向观察方向 | 估值/预期低位，但主流资金、广度和热度未确认 | 只能低关注跟踪 |
| 状态不明 | 数据不足、来源冲突或超过有效期 | 重新计算后再推荐 |

热门方向不是永久热门；低活跃方向也不是永久排除。所有状态都必须由账本证据或当期数据支持。

## 4. 推荐读取规则

个股推荐 skill 必须按以下顺序使用本账本：

1. 读取最近有效的运行态板块状态，而不是把本规则文件当作当前市场数据。
2. 先按短线主线规则选出 2-3 条方向：涨幅/相对强度、成交热度、广度、资金、催化至少 3 项成立，且资金或成交热度必须至少 1 项成立。
3. 再从这些主线内找业绩/基本面质量较好、资金结构不差、短期涨幅未透支的个股。
4. 避免从 `低活跃方向` 中默认选股。
5. 如果用户要求逆向/左侧机会，允许读取 `逆向观察方向`，但不得给高关注等级。
6. 如果账本缺失、过期或和当日行情冲突，必须重新计算相关板块状态，并在输出中说明。

短线主线不是简单追涨规则：

- 单日涨幅强但资金流出、成交断层或只有后排补涨时，不得列为高优先级主线。
- 资金净流入强但缺少产业/政策/业绩传导时，只能列入观察，不直接进入高关注推荐。
- 主线内最强的一字板、快速封板和连续涨停股用于证明市场强度，进入 `sector_anchors`，不进入推荐前 5。
- 进入推荐清单的标的应来自同一主线中相对未透支、仍有成交窗口、业绩或经营质量能支撑逻辑的公司。

## 6. 质量要求

- 热门方向必须有持续性证据，不能只因单日涨幅或新闻热搜入选。
- 低活跃方向必须由最近一段时间数据证明，不能凭行业刻板印象写入。
- 所有状态必须保留数据时间、来源和口径。
- 状态账本只影响推荐池优先级，不替代个股基本面、估值、风险和来源核验。
- 不得把仓库模板里的示例状态当作当前真实市场状态。
- 热门方向必须检查过热风险；低活跃方向必须检查上调触发条件。
