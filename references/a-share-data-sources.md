# A 股数据源与降级策略

本文件定义龙虾投资助手的 A 股数据获取优先级、最小核验要求和失败降级方式。所有涉及高时效金融事实的 skill 都应先读取本文件，再执行具体分析。

## 1. 数据类型与首选来源

| 数据类型 | 首选来源 | 可接受备选 | 最小要求 |
|---|---|---|---|
| 股票代码、证券简称、交易所 | 上交所、深交所、北交所、东方财富、同花顺 | 证券时报、财联社 | 必须确认唯一标的 |
| 实时/最近行情 | 交易所行情页、东方财富、同花顺、TradingView | 新浪财经、腾讯财经 | 必须标注数据时间 |
| 公司公告 | 巨潮资讯、交易所公告、上市公司官网 IR | 主流财经媒体转载 | 重大结论优先用原始公告 |
| 财务报表 | 巨潮资讯定期报告、交易所披露、公司年报/季报 | 东方财富财务、同花顺财务 | 必须标注报告期 |
| 估值指标 | 东方财富、同花顺、Wind/Choice 可访问页 | 交易所、市值行情源 | 必须说明口径和时间 |
| 资金流 | 沪深港通、交易所融资融券、东方财富资金流、同花顺资金流 | 财联社、证券时报报道 | 必须标注统计区间 |
| 龙虎榜 | 上交所、深交所、东方财富龙虎榜 | 同花顺龙虎榜 | 必须标注交易日期 |
| 宏观/行业政策 | 国务院、发改委、工信部、财政部、央行、证监会 | 新华社、证券时报、财联社 | 重大政策优先官方来源 |
| 外围市场/商品 | CME、LME、ICE、EIA、OPEC、主要指数官网 | 路透、彭博、东方财富国际行情 | 必须说明关联链路 |
| 板块热度/行业强弱 | 申万/中信行业指数、交易所行业分类、东方财富行业/概念板块、同花顺板块、ETF 行情 | 财联社、证券时报、主流券商策略观点 | 必须标注统计区间、涨跌幅/成交额/资金流/涨跌家数等口径 |

## 1.0.0 本仓库 Provider Registry

本仓库脚本层按 provider registry 管理免费数据源，不把任何单一免费网页接口当作绝对稳定来源。

| Provider | 当前用途 | 稳定性分层 | 接入方式 | 使用规则 |
|---|---|---|---|---|
| `eastmoney` | 个股行情、行业/概念板块快照 | S3 | 直接调用东方财富公开网页接口 | 默认主源；必须保留字段、数据时间、访问时间和限制说明 |
| `tencent` | 个股行情交叉校验和兜底 | S4 | 直接调用腾讯公开网页行情接口 | 不单独支撑高确定性推荐；用于与东方财富核对价格 |
| `akshare_ths` | 同花顺行业/概念板块快照 | S3 | 可选安装 AKShare 后调用同花顺相关封装接口 | 不视为同花顺官方稳定 API；用于板块热度补充和交叉观察 |

东方财富和同花顺的判断：

- 东方财富公开网页接口覆盖行情、行业板块和概念板块，字段较完整，适合作为免费 S3 主源，但接口参数和字段没有官方长期稳定承诺。
- 同花顺公开侧没有适合本仓库直接硬编码的免费官方稳定 API；需要同花顺数据时优先通过 AKShare 这类维护中的封装层，或使用付费/终端/正式授权数据。
- 对关键推荐结论，不得只依赖 `eastmoney` 或 `akshare_ths` 单一板块快照；至少结合运行态板块账本、行情交叉校验、催化来源和风险降级。

脚本入口：

```bash
python3 scripts/fetch_a_share_data.py 000001 600519 --providers eastmoney,tencent
python3 scripts/fetch_sector_boards.py --provider eastmoney --kind concept
python3 scripts/fetch_sector_boards.py --provider akshare_ths --kind industry
```

`akshare_ths` 是可选能力；没有安装 AKShare 时脚本必须记录错误并降级，不能编造同花顺数据。

## 1.0 稳定性分层

所有数据源按稳定性分层使用。结论越关键，越应该使用高稳定性来源；低稳定性来源只能辅助，不得单独支撑核心推荐。

| 层级 | 类型 | 例子 | 使用方式 |
|---|---|---|---|
| S1 原始/官方来源 | 信息发布主体或监管披露 | 交易所、巨潮资讯、上市公司公告、监管机构、政府部门 | 公司公告、财报、政策、交易规则必须优先使用 |
| S2 结构化数据源 | 有相对稳定 SDK/API/数据模型 | BaoStock、TuShare Pro、AKShare 中的历史 K 线/财务接口、交易所行情下载 | 行情、历史收盘、交易日历、财务字段优先使用 |
| S3 主流行情/财经平台 | 可核验但接口可能变化 | 东方财富、同花顺、雪球、证券时报、财联社 | 实时行情、资金流、新闻催化可用，但必须标注时间和口径 |
| S4 网页抓取/非正式接口 | 无官方稳定承诺 | 新浪/腾讯网页行情、网页 JSON 接口、第三方封装 | 只能作补充或交叉校验，不作为唯一关键来源 |
| S5 社交/自媒体/传闻 | 非原始、不可稳定核验 | 论坛、群聊、短视频、未具名自媒体 | 不得作为核心事实来源 |

稳定数据源优先级：

1. 公告、财报、政策：必须先找 S1；没有 S1 时，只能写“未取得原始来源”，不得下确定结论。
2. 09:00 定时任务价格：优先用 S2 的最近交易日历史日线或交易日历确认，不能用 S3/S4 的实时字段冒充当前价。
3. 盘中行情：允许使用 S3/S4，但必须保留 `data_time`，并尽量用两个来源交叉校验。
4. 估值和财务：优先 S1/S2；S3 平台估值必须写明平台、数据时间和指标口径。
5. 单一 S4 来源只能输出低确定性事实；涉及推荐理由、目标价、估值判断时必须有 S1/S2/S3 支撑。

## 1.0.1 数据整理不偏离原文

从来源整理成结构化数据时，必须保留原始事实和归一化过程，不能为了让结论更顺而改写数据含义。

每个关键字段至少记录：

- `raw_value`：来源中的原始值或原文摘要。
- `normalized_value`：整理后的值。
- `unit`：单位，如元、亿元、%、万股、手。
- `data_time`：数据时间、公告日期或报告期。
- `source_field`：来源页面/接口中的字段名或原文位置。
- `transform`：换算方式，如万元转亿元、百分比转小数、前复权/不复权。
- `limitations`：口径限制、延迟、缺失项或不确定性。

禁止事项：

1. 不得把“同比下降收窄”整理成“同比增长”。
2. 不得把“预测/一致预期/机构目标价”整理成真实财务或行情数据。
3. 不得省略单位换算，尤其是元/万元/亿元、股/手、人民币/港币/美元。
4. 不得把旧报告期数据写成最新报告期。
5. 不得把媒体转述整理成公司公告事实，除非能追溯到原公告。
6. 不得在多个来源数值不一致时自行取一个更符合结论的数；必须说明冲突和采用理由。

## 1.1 交易时段价格口径

A 股定时任务必须根据运行时间标注价格口径，不能把非交易时段数据写成盘中实时价。

| 运行时间 | 允许价格口径 | 必填说明 |
|---|---|---|
| 09:00 前后、09:15 前 | 上一交易日收盘价 / 最近可核验价 | 写明“截至上一交易日收盘”，不得写“当前价” |
| 09:15-09:25 | 集合竞价参考价 | 写明“集合竞价参考，非最终成交价” |
| 09:25-09:30 | 开盘前参考价 / 上一交易日收盘价 | 写明数据源时间，避免当作连续竞价实时价 |
| 09:30-11:30、13:00-15:00 | 盘中实时价或延迟行情价 | 写明数据源、数据时间、是否延迟 |
| 11:30-13:00 | 午间休市最新成交价 | 写明“午间休市，截至上午收盘/最近成交” |
| 15:00 后 | 当日收盘价或盘后最近价 | 写明“截至当日收盘” |
| 非交易日 | 最近交易日收盘价 | 写明最近交易日日期 |

如果行情接口返回的数据时间不在上述有效时段内，必须按“最近可核验价”处理，并在输出中说明限制。

## 1.1.1 板块热度与行业强弱口径

个股推荐必须先判断所属行业/主题是否处在可推荐的市场环境中，不能只因为个股估值低、故事完整或历史龙头地位就纳入高优先级推荐。

推荐前应按 `references/sector-state.md` 中定义的账本规则读取运行态板块状态。热门方向和低活跃方向都应来自最近一段时间的定时任务、盘后复盘或周度刷新沉淀，不应每次完全凭即时印象重算，也不应在仓库中形成永久静态名单。

板块热度至少核验以下 4 类中的 3 类：

1. 区间表现：最近 1 日、5 日、20 日涨跌幅或相对沪深 300/中证 全指的超额收益。
2. 交易活跃度：成交额、成交额环比、换手率、量比或行业 ETF 成交变化。
3. 广度：板块内上涨家数占比、创新高家数、强势股数量、跌破均线家数。
4. 催化：政策、订单、产品价格、产业数据、海外映射、财报验证或重大事件。
5. 资金：主力资金、北向、融资融券、ETF 份额或龙虎榜席位变化。

推荐门槛：

- 高优先级推荐必须来自“强势板块”“景气改善板块”或“明确催化板块”。
- 当期数据判定为低活跃的方向，只有在出现可核验催化、成交放大、板块相对强度改善时才允许推荐；否则放入“不推荐但跟踪”。
- 热门方向必须有最近 5 个交易日左右的持续证据，不能只因单日涨幅、热搜或单条新闻直接判定。
- 低估值不能单独成为推荐理由；必须叠加行业景气改善、政策/订单/价格催化或资金回流证据。
- 如果板块热度数据缺失，不能给高关注等级；最多输出“低确定性推荐”或要求补充行情源。
- 如果用户明确要求逆向/左侧机会，必须把推荐类型标成“逆向观察”，并说明主流资金和板块热度尚未确认。

可参与性门槛：

- 开盘即涨停、一字板、开盘后快速封板的标的，不得作为普通投资者的当日推荐前 5，只能作为板块强度锚点。
- 近 5 个交易日涨停次数 `>=2` 的标的，默认筹码拥挤，必须从推荐清单降级为主线锚点或暂不推荐。
- 近 10 个交易日涨幅 `>=35%` 的标的，默认追高风险过高，不进入前 5 推荐；除非用户明确要求高风险短线研究，也只能写成高风险观察。
- 推荐链路应优先从同一热门主线里寻找尚未极端加速、成交可参与、分歧后承接仍在的公司，而不是直接推荐最强涨停龙头。

## 1.2 估值数据硬规则

估值数据必须比行情价更严格处理，因为同一个“PE/目标价/估值分位”可能来自不同口径。

1. 估值指标必须写明具体口径：PE-TTM、PE-Forward、PB、PS、EV/EBITDA、市值、股息率、历史分位或同行分位，不能只写“估值高/低”。
2. 每个估值数据必须有来源、数据时间和报告期；财报口径至少说明使用年报、季报、TTM 还是一致预期。
3. 净利润为负或一次性损益扭曲时，不得直接使用 PE 判断贵便宜；应改用 PB、PS、EV/收入、现金流、分部估值或明确写“PE 失真”。
4. 历史分位和同行对比不能凭感觉生成；没有可核验来源时，只能写“未取得可靠历史分位/同行分位”。
5. 外部目标价或一致预期必须写明机构/平台、发布日期、估值方法和链接；超过 90 天的目标价默认降级为旧参考。
6. 模型预测价必须显式标注为“预测值”，写明核心假设、计算口径和失效条件，不得包装成确定目标价。
7. A/H 股、不同币种、复权/不复权、市值口径、股本口径不得混用；如口径不一致，必须先换算或放弃比较。
8. 估值结论只能表述为“相对自身历史/同行/盈利预期偏高、偏低或合理”，不得直接推出买入、卖出、加仓、清仓。

Quality Checklist：

- 估值指标是否写明来源、数据时间、报告期和计算口径。
- 分母是否有效：利润为负、一次性损益、周期峰谷、股本变化是否已处理。
- 估值结论是否区分真实数据、外部参考和模型预测，并避免交易指令。

## 2. 最小证据门槛

### 个股分析

至少需要：

- 标的身份：证券简称、代码、交易所。
- 行情：最近交易日或实时行情，含数据时间。
- 公司基本面：主营或收入结构。
- 财务：最近一期营收、利润、现金流或毛利率中的至少 2 项。
- 事件：用户问题相关公告、新闻或明确说明未查到。
- 风险：至少 2 条与公司或行业相关的风险。

缺少任一关键项时，必须在结论中降低确定性。

### 候选观察池

每只候选至少需要：

- 证券简称、代码、交易所。
- 行情数据时间。
- 主营与行业位置。
- 最近一期财务或估值指标。
- 关注理由、主要风险、逻辑失效条件。
- 至少 2 个可追溯来源。

来源不足时，不得给“高关注”。

### 财报/公告解读

至少需要：

- 公告标题、披露日期、披露主体。
- 原始公告链接或交易所/巨潮链接。
- 关键事实摘取。
- 对收入、利润、现金流、资产负债、估值或风险偏好的影响链路。
- 反向风险或尚未确认事项。

## 3. 运行时检查建议

在 CLI/本地 agent 环境中，优先检查这些能力：

```bash
command -v python3 >/dev/null && echo PYTHON_OK || echo PYTHON_MISSING
command -v curl >/dev/null && echo CURL_OK || echo CURL_MISSING
command -v jq >/dev/null && echo JQ_OK || echo JQ_MISSING
```

如果存在本地金融数据 CLI 或 MCP，应先检查其状态；如果不存在，使用网页搜索和官方链接完成核验。

## 4. 降级策略

| 失败情况 | 处理方式 |
|---|---|
| 找不到唯一股票代码 | 先澄清，不分析 |
| 行情源不可用 | 使用最近可核验行情，并明确非实时 |
| 09:00 定时任务取不到当日盘中价 | 使用上一交易日收盘价，并明确“非实时、截至上一交易日收盘” |
| 行情接口返回时间早于任务运行日 | 使用最近可核验价，并输出数据时间，不得写当前价 |
| 原始公告打不开 | 使用交易所/巨潮备用入口；仍打不开则不能做确定结论 |
| 财务数据源冲突 | 以定期报告或交易所披露为准 |
| 资金数据缺失 | 明确“资金面未核验”，不得编造资金流 |
| 新闻只有传闻 | 标注未核验，默认不纳入核心判断 |
| 来源无法支撑结论 | 删除该结论或改成待确认 |

## 5. 禁止事项

- 不得把搜索结果页当作来源。
- 不得把自媒体、群聊、论坛传闻当作事实。
- 不得用旧公告冒充最新公告。
- 不得混淆 A 股、港股、美股同名公司。
- 不得在来源不足时输出确定性强的结论。
