# 龙虾投资助手工作流程

## 1. 总体任务路由

收到用户问题后，先判断任务类型：

| 用户意图 | 使用内容 |
|---|---|
| 整理 A 股核心财经要闻 | `skills/a-share-market-news/SKILL.md` |
| 分析单只或多只个股 | `skills/a-share-stock-analysis/SKILL.md` |
| 推荐股票/筛选候选观察池 | `skills/a-share-stock-recommendation/SKILL.md` |
| 审计自身回答 | `self-improvement/INDEPENDENT_REVIEW_PROMPT.md` |
| 基于 Review 结果提出优化 | `self-improvement/OPTIMIZATION_ADVISOR_PROMPT.md` |

## 2. 财经要闻流程

适用场景：

- 今日 / 昨日 / 近 1 日 / 近 3 日 / 近一周
- 上周最后一个交易日
- 盘前/盘后利好利空

执行：

1. 明确时间范围。
2. 联网核验官方、交易所、公司公告、主流财经源。
3. 剔除软文、重复资讯、口水新闻、传闻。
4. 按利好/利空分块。
5. 每条输出：核心摘要、对应板块/相关个股、涨跌影响逻辑。

## 3. 个股分析流程

适用场景：

- 某股票怎么看
- 为什么涨/跌
- 公告是否利好/利空
- 基本面、估值、资金面分析

执行：

1. 确认证券简称、代码、交易所。
2. 核验最新行情和数据时间。
3. 核验主营业务、财务指标、估值、公告、资金、行业催化。
4. 区分短线、中期、长期逻辑。
5. 给出关键观察点、主要风险、逻辑失效条件。
6. 来源链接集中附在最后。

## 4. 个股候选推荐流程

适用场景：

- 推荐几只股票
- 找潜在机会
- 生成候选池
- 按条件选股

执行：

1. 明确市场范围、风格偏好、行业范围、时间维度、风险偏好。
2. 默认输出 3-5 只候选观察标的。
3. 每只股票至少核验行情、主营、财务、估值、催化、风险。
4. 输出关注等级：高关注 / 中关注 / 仅观察 / 暂不纳入。
5. 必须说明逻辑失效条件。
6. 不给买入价、目标价、止损价、仓位比例。

## 5. 自我迭代流程

按三角色执行：

```text
任务执行者 -> 独立 Reviewer -> 评分与优化建议者 -> 人工确认 -> 回归测试
```

禁止：

- 执行者自己给自己评分。
- 没有独立 Review 报告就给通过结论。
- 用最终答案外观替代来源核验。
